AI assessment settings
Model Claude Sonnet 4  (claude-sonnet-4-20250514, Anthropic)
Temperature 0.3 — Controls how consistent the AI’s responses are. At 0, the model always gives the same answer for the same input. At 1 (the default), it varies more. Set to 0.3 to reduce verdict variability on repeated assessments of the same note, while retaining the flexibility needed for clinical reasoning.
Documentation standard Strict — only explicitly stated facts count, mirroring how a triage radiologist reads a referral cold. Inferred — reasonable clinical context is allowed (e.g. “No AF” — no atrial fibrillation — suggests anticoagulation for AF is less likely, though other indications may exist). Selectable per assessment.
Prompt caching The ~21,000-token criteria block is cached server-side after the first call of a session, reducing cost and latency on subsequent calls within that session.
Important limitations
  • This tool is a prototype. It has not been validated for clinical use.
  • Criteria reflect the August 2025 reissue. If the national document is updated, this tool must be updated separately — it does not update automatically.
  • AI assessments are probabilistic and may occasionally produce different results for the same note.
  • Always verify against the source document before submitting a referral.
  • Do not enter identifiable patient information — note text is sent to the Anthropic API for assessment.
Criteria update process
Criteria updates are the responsibility of Health New Zealand | Te Whatu Ora. When the national criteria document is revised, HNZ must initiate an update to this tool by providing the updated PDF to the tool developer. The structured criteria data will then be re-extracted, diffed against the current version for clinical review and approval by HNZ, and a new tool version will be produced. The UI and AI assessment logic are independent of the criteria data — a criteria update does not require rebuilding the tool. The criteria version and date are always shown in this panel.
